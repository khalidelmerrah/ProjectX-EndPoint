Privacy Policy

Effective Date: December 26, 2025

ProjectX is designed with a fundamental commitment to user privacy and data sovereignty. We define our architecture as "Local-First". This means the software prioritizes storing and processing data on your local machine over cloud servers.

1. Data Collection & Telemetry

ProjectX does NOT collect, transmit, or sell personal data.

  • No Usage Analytics: We do not track button clicks, session duration, or user behavior.
  • No Crash Reporting: Stack traces are logged locally to %APPDATA%/ProjectX/logs. They are not uploaded to us.
  • Local Database: All inventory data (software lists, process monitoring logs, scan results) is stored in a SQLite database (`projectx.db`) located strictly on your local filesytem.

2. Third-Party Integrations

ProjectX includes optional integrations with third-party services. These features are disabled by default and require you to manually provide your own API keys.

2.1 NIST National Vulnerability Database (NVD)

If enabled, the application contacts NIST servers to download critical vulnerability information (CVEs). Data Sent: None (Read-Only download).

2.2 Google Gemini AI (Generative Analysis)

If provided with an API Key, you may choose to click "AI Analyze" on specific alerts.
Data Sent: Only the specific text snippet you chose to analyze (e.g., a process name or log line) is sent to Google's API.
Storage: Please refer to Google's Generative AI Privacy Terms.

2.3 VirusTotal

Data Sent: File Hashes (SHA-256) only. We do not upload file contents.

3. Secret Storage

ProjectX does NOT store API keys, passwords, or secrets in plain-text configuration files. All secrets are stored in the operating system's native vault:
Windows: Windows Credential Locker

4. Contact

For privacy inquiries or to report a security vulnerability, please open an issue on our GitHub Repository.
